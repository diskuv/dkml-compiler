# byterntm dependencies

The Windows 4.14.3 build needed a fix because `make coreall -j4` could compile `bytecomp/byterntm.cmi` before `utils/misc.cmi`, failing with `Error: Unbound module Misc` (patch `a63` makes `bytecomp/byterntm.mli` reference `Misc.RuntimeID.t`). The relocatable-backport `byterntm` module is absent from the committed `.depend`, so `ocamldep` never records its dependencies and parallel `make` is free to order it too early. The fix adds the missing `Makefile` dependencies so `bytecomp/byterntm.cmi` waits for `utils/misc.cmi`, and `bytecomp/byterntm.cmo`/`bytecomp/byterntm.cmx` wait for `bytecomp/byterntm.cmi`, `bytecomp/bytesections.{cmi,cmx}`, and `utils/misc.{cmi,cmx}`.
